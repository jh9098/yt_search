# PLAN.md

## 1. 프로젝트명
유튜브 소재 채굴기 + AI 소재 분석 (MVP)

---

## 2. 프로젝트 목적
유튜브 영상/채널 데이터를 기반으로 콘텐츠 제작자가 빠르게 소재를 발굴하고,
AI 분석을 통해 댓글 반응/긍정 포인트/아쉬운 점/확장 가능한 후속 소재를 얻도록 돕는 웹 서비스를 만든다.

---

## 3. 핵심 사용자
- 유튜브 콘텐츠 제작자 (정보성/교육형/심리/건강/경제 등)
- 소재 조사 시간을 줄이고 싶은 1인 운영자
- AI로 기획/대본 초안을 빠르게 만들고 싶은 사용자

---

## 4. MVP 범위 (이번 단계에서 만들 것)
### 4.1 검색/탐색
- 키워드 검색
- 기본 필터(기간/최소 조회수/영상 길이/정렬 등 일부)
- 영상 카드 목록 표시

### 4.2 AI 소재 분석 모달
- "AI 소재 분석" 버튼 클릭
- 모달 열기
- 분석중 상태 표시 (loading)
- 분석완료 상태 표시 (success)
- 실패 상태 표시 (error)
- 분석 결과 섹션 표시:
  - 댓글 요약
  - 주요 반응
  - 긍정 포인트
  - 아쉬운 점
  - 소재 추천
  - 추천 키워드

### 4.3 분석 결과 저장(초기)
- 동일 영상 재분석 시 캐시 재사용 가능하도록 구조 설계
- 최소한 결과 저장 테이블/스키마 설계

---

## 5. MVP 제외 범위 (지금은 하지 않음)
- 결제/구독 시스템
- 사용자 권한/팀 협업
- 고급 추천 알고리즘
- 실시간 알림
- WebSocket 실시간 진행률 (초기엔 polling 우선)
- 대규모 자동 수집 파이프라인 고도화
- 완전한 운영 관리자 페이지

---

## 6. 기술 스택 (초기 가정)
### Frontend
- React + Vite + TypeScript + Tailwind CSS

### Backend
- FastAPI + Pydantic

### Infra/Data
- PostgreSQL
- Redis (추후 또는 초기 일부)
- Render / Netlify 계열 배포 고려

### AI
- Gemini API (분석 결과 생성)

---

## 7. 핵심 설계 원칙
1. 응답 스키마 우선
2. 상태 전환 명확화 (`loading/success/error`)
3. 에러/빈값 처리 우선
4. 캐시 고려 (중복 비용 절감)
5. 문서(계획/맥락/체크리스트) 우선 운영

---

## 8. 사용자 흐름 (MVP)
1. 사용자가 키워드 검색
2. 영상 카드 목록 확인
3. 특정 영상에서 "AI 소재 분석" 클릭
4. 모달에서 분석중 상태 확인
5. 분석완료 후 인사이트 확인
6. 추천 키워드 클릭 → 재검색 (후속)
7. (추후) 백로그 저장 / 대본 생성 연결

---

## 9. 단계별 개발 계획 (초안)
### Phase 1: 문서/설계 세팅
- AGENT/SKILLS/3대 문서 생성
- API 응답 형식 초안
- AI 출력 스키마 초안
- 모달 상태 설계

### Phase 2: 프론트 UI (가짜 데이터)
- 카드 목록 UI
- 분석 모달 loading/success/error UI
- 추천 키워드 칩 UI

### Phase 3: 백엔드 API 기본
- 분석 job 생성/조회 API (또는 임시 동기 버전)
- 응답 스키마 정의
- 에러 응답 규칙 정리

### Phase 4: AI 연동 + 캐시
- Gemini 호출
- 출력 JSON 스키마 검증
- 결과 저장/캐시
- 재시도/실패 처리

### Phase 5: 연결/테스트
- 프론트-백엔드 연결
- 예외 케이스 테스트
- 문서 업데이트

---

## 10. 완료 기준 (Definition of Done for MVP 1차)
아래 조건을 만족하면 1차 MVP 완료로 간주:
- [ ] 검색 결과 카드에서 "AI 소재 분석" 버튼 클릭 가능
- [ ] 모달에 loading/success/error 상태가 모두 구현됨
- [ ] 백엔드 분석 응답 스키마가 고정되어 있음
- [ ] AI 분석 결과가 최소 1개 샘플 영상 기준 정상 표시됨
- [ ] 실패 시 사용자에게 재시도/안내 메시지 제공
- [ ] 체크리스트/맥락노트/결정사항 문서가 업데이트됨

---

## 11. 추정치/불확실성 표시 정책
다음은 사용자에게 "추정치"임을 명시해야 한다.
- 예상 수익
- 성과도/기회점수 등 내부 계산 점수
- 일부 AI 추론 기반 분류 결과

표시 예시:
- "추정치"
- "AI 제안"
- "내부 계산 지표"
