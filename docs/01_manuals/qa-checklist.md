# qa-checklist.md

## 목적
작업 완료 전에 AI/개발자가 빠뜨리기 쉬운 항목을 점검하기 위한 셀프 체크리스트.

---

## 1. 요구사항/범위 체크
- [ ] 이번 작업 범위가 `PLAN.md`와 일치하는가?
- [ ] 임의로 기능을 추가/확장하지 않았는가?
- [ ] 제외 범위를 침범하지 않았는가?
- [ ] 변경된 설계가 있다면 `CONTEXT_NOTE.md` 또는 `DECISIONS.md`에 기록했는가?

---

## 2. 코드 품질 체크
- [ ] 수정 코드가 과도하게 길어지지 않았는가? (분리 필요성 검토)
- [ ] 변수/함수/컴포넌트 이름이 역할을 설명하는가?
- [ ] 하드코딩 값/매직 스트링이 남아있지 않은가?
- [ ] 불필요한 콘솔 로그/디버그 코드가 남아있지 않은가?

---

## 3. 프론트엔드 체크 (해당 시)
- [ ] `loading/success/error` 상태가 모두 처리되었는가?
- [ ] API 실패 시 사용자에게 안내/재시도 경로가 있는가?
- [ ] null/undefined/빈 배열 케이스에서 UI가 깨지지 않는가?
- [ ] 버튼 중복 클릭 방지(로딩 중 disabled 등)가 있는가?
- [ ] 긴 콘텐츠(모달/패널) 스크롤 처리가 적절한가?

---

## 4. 백엔드 체크 (해당 시)
- [ ] 요청 입력값 검증이 있는가?
- [ ] 응답 스키마가 문서/API 계약과 일치하는가?
- [ ] 예외 처리(외부 API 실패, timeout, 빈 데이터)가 있는가?
- [ ] 외부 API 호출 timeout이 설정되어 있는가?
- [ ] 로그에 민감정보(API key, token)가 노출되지 않는가?

---

## 5. 데이터/AI 분석 체크 (해당 시)
- [ ] AI 출력이 JSON 스키마 검증을 통과하는가?
- [ ] 필드 누락 시 기본값 처리 정책이 있는가?
- [ ] "사실 요약"과 "AI 제안"이 혼동되지 않게 구조화했는가?
- [ ] 분석 메타데이터(모델명/분석시각/기준 데이터 수)를 기록하는가?
- [ ] 추정치(예상 수익/점수 등)를 추정치로 표시했는가?

---

## 6. 테스트 체크
- [ ] 정상 케이스 테스트 완료
- [ ] 실패 케이스 테스트 완료 (네트워크/API 실패 등)
- [ ] 빈 데이터 케이스 테스트 완료
- [ ] 경계값/이상값 케이스 확인 (짧은 제목, 댓글 없음 등)

---

## 7. 문서/기록 체크
- [ ] `CHECKLIST.md` 업데이트 했는가?
- [ ] 필요 시 `CONTEXT_NOTE.md` 업데이트 했는가?
- [ ] 필요 시 `DECISIONS.md` 업데이트 했는가?
- [ ] 완료 보고서 템플릿 형식으로 결과를 정리했는가?

---

## 8. 최종 완료 선언 전
- [ ] "다 했습니다" 한 줄이 아니라, 변경 내용/이유/테스트/리스크를 보고 형식으로 정리했는가?
- [ ] 다음 작업이 무엇인지 제안했는가?
