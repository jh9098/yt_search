AI와 함께하는 **바이브 코딩(Vibe Coding)**은 속도는 빠르지만, 대화가 길어질수록 AI가 앞선 내용을 잊어버리는 '금붕어 기억력'이 가장 큰 걸림돌입니다. 이를 극복하고 끊김 없이 개발을 이어가기 위해 소스에서 강조하는 **'기억 시스템 세팅 및 개발 프로세스'**를 단계별로 설명해 드립니다.
1. 하니스(Harness) 및 매뉴얼 세팅 (기본 환경 구축)
가장 먼저 AI가 언제든 참조할 수 있는 업무 지침서를 만들어야 합니다. 이는 AI에게 출근하자마자 읽어야 할 '매뉴얼'을 주는 것과 같습니다.
• agent.md 또는 skills.md 생성: 프로젝트의 전반적인 규칙, 사용할 기술 스택(예: Next.js + FastAPI), 선호하는 디자인 스타일(예: 미니멀, Shadcn UI) 등을 이 파일에 미리 적어두세요.
• 훅(Hooks) 시스템 활용: AI가 작업을 시작하기 전에 자동으로 매뉴얼을 읽고, 작업이 끝나면 품질을 검사하도록 강제하는 '자동 알림 장치'를 설정하는 것이 좋습니다.
2. AI의 외부 기억장치: 3대 핵심 문서 만들기
AI가 대화창을 새로 열거나 맥락을 놓치더라도 즉시 복구할 수 있도록 프로젝트 폴더 내에 3가지 문서를 파일로 남겨야 합니다.
1. 계획서 (Plan): 우리가 무엇을 만들 것인지 전체적인 설계도를 적습니다.
2. 맥락 노트 (Context Note): 왜 이런 결정을 내렸는지, 특정 로직이 왜 그렇게 짜였는지 배경 정보를 기록합니다.
3. 할 일 체크리스트 (Checklist): 무엇을 완료했고, 다음에 무엇을 해야 하는지 진행 상황을 추적합니다. AI가 한 단계를 끝낼 때마다 이 목록을 직접 업데이트하게 시키면 성능이 가장 좋아집니다.
3. 바이브 코딩 실전 프로세스 (까먹지 않는 법)
기획이 완료된 상태에서 개발을 시작할 때 다음의 '계획-정지-실행' 루틴을 따르세요.
• 단계별 지시와 승인: 한 번에 전부 만들라고 하지 말고, 기능을 잘게 나누어 시키세요.
• "잠깐 멈춰" 전략: AI에게 계획을 짜게 한 뒤, 바로 코딩에 들어가지 못하게 하세요. **"좋아, 이 계획을 먼저 문서(3대 핵심 문서)로 저장해"**라고 지시한 뒤, 저장된 파일을 기반으로 작업을 시작하게 하는 것이 핵심입니다.
• DDD(도메인 주도 설계) 적용: 코드를 비즈니스 개념(예: 유저, 결제, 게시판) 단위로 폴더를 나누어 관리하세요. 이렇게 하면 AI에게 특정 기능을 고쳐달라고 할 때 해당 폴더의 파일만 보여주면 되므로 AI의 집중력이 올라가고 환각(Hallucination)이 줄어듭니다.
4. 주의사항 및 보안 체크
AI는 코드를 빠르게 짜지만, 보안이나 데이터 관리에서 실수를 자주 범합니다.
• 보안 취약점 확인: 결제 알림 검증, 로그인 권한 관리, 데이터베이스 이중 결제 방지 로직 등은 AI가 습관적으로 구멍을 내는 영역이므로 사람이 반드시 최종 확인해야 합니다.
• 주기적인 세션 초기화: 대화 기록이 너무 길어지면 AI가 느려지고 실수가 잦아집니다. 하나의 작업 단위(예: 자료 조사, UI 구현)가 끝나면 새 대화창을 열고 위에서 만든 '기억 문서'들을 다시 읽게 하여 작업을 이어가는 것이 훨씬 안정적입니다.
요약하자면, AI를 믿지 말고 AI가 참고할 **외부 문서(계획/맥락/체크리스트)**를 파일로 관리하며, 도메인별로 코드를 구조화하는 것이 바이브 코딩의 성공 비결입니다.
추가적으로, 소스에서 추천하는 Next.js(프론트)와 Fast API(백엔드) 조합을 사용하면 AI가 학습한 데이터가 많아 훨씬 정확한 코드를 얻으실 수 있습니다. 가이드가 필요하시다면 '타일러드 리포트' 생성을 요청해 주세요. 모든 소스를 기반으로 더 상세한 개발 가이드를 만들어 드릴 수 있습니다. 생성해 드릴까요?
